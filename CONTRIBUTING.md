# Contributing to the Project

First off, thank you for considering contributing! Your help is appreciated. This document provides a set of guidelines for contributing to this project.

## Code of Conduct

This project and everyone participating in it is governed by a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs
- Use the **Bug Report** issue template to ensure your report is complete.
- Provide a clear title and a detailed description of the issue.
- Describe the exact steps to reproduce the bug.
- Include screenshots or screen recordings if possible.

### Suggesting Enhancements
- Use the **Feature Request** issue template.
- Provide a clear and detailed explanation of the feature you're suggesting.
- Explain why this enhancement would be useful.

### Pull Requests
1.  **Fork the repository** and create your branch from `main`.
2.  **Name your branch** according to the convention: `[type]/[ticket-number]-[description]` (e.g., `feat/T-123-add-dark-mode`).
3.  **Set up your development environment** by following the [Installation Guide](./README.md#installation-guide).
4.  **Make your changes.** Ensure your code adheres to the project's code style.
5.  **Update documentation** (`README.md`, component comments, etc.) with any changes that require it.
6.  **Run the linter** to check for issues: `npm run lint`.
7.  **Commit your changes** using a clear and descriptive commit message.
8.  **Push your branch** to your fork and open a pull request to the main repository.
9.  **Fill out the Pull Request template** completely.

## Code Style
- This project uses **ESLint** for code analysis and **Prettier** for formatting.
- Please run `npm run lint:fix` before committing to automatically format your code and fix any simple linting issues.

Thank you for your contribution!
